# UI デザインルール

UI/フロントエンドの実装時に適用する原則。

## 基本原則

- **見せてから作る** — AI はコピーが得意で想像が苦手。ビジュアル参照なしに UI を実装しない
- **デザインシステム先行** — コード実装前にカラー、タイポグラフィ、スペーシング、アイコンを定義する
- **一貫性を最優先** — 全ページ・全コンポーネントでデザインシステムに従う

## ビジュアル参照の要求

UI 実装の指示を受けたとき、以下のいずれかが提供されていなければ確認する:

- ワイヤーフレーム（Excalidraw、手書きスケッチ等）
- 参考スクリーンショット（Dribbble、Mobbin、既存サイト等）
- ムードボード（配色の方向性）
- 既存のデザインシステム定義

「モダンにして」「きれいにして」のような曖昧な指示だけでは実装に入らない。具体的なビジュアル参照を求める。

## アンチパターン（禁止事項）

- Inter + Lucide のデフォルト組み合わせをそのまま使わない
- 根拠のない AI 的グラデーション（青→紫の典型パターン）を使わない
- デザインシステム未定義のまま複数ページを実装しない
- 「なんとなくいい感じに」で色やフォントを選ばない

## デザインシステムの最小定義

実装前に以下を明確にする:

| 項目 | 内容 |
|------|------|
| カラーパレット | Primary, Secondary, Accent, Background, Text, Error, Success |
| タイポグラフィ | フォントファミリー、サイズスケール（h1〜body〜caption） |
| スペーシング | 基本単位（4px/8px）とスケール |
| 角丸 | ボタン、カード、入力フィールドの統一値 |
| アイコン | 使用ライブラリ（Phosphor、Heroicons 等。Lucide 以外を検討） |
| シャドウ | エレベーション段階（sm, md, lg） |
