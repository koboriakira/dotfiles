# UI 実装ワークフロー

UI/フロントエンドの実装を開始するときに使用する。デザインシステム先行・ビジュアル参照必須の原則を強制する。

## 実行フロー

### 1. ビジュアル参照の収集

ユーザーに以下を確認する:

```
## ビジュアル参照の確認

実装に入る前に、参照素材を集めます。

| # | 種類 | 説明 |
|---|------|------|
| 1 | ワイヤーフレーム | Excalidraw や手書きスケッチの画像 |
| 2 | 参考スクリーンショット | 真似したいサイトやUIの画像 |
| 3 | ムードボード | 配色や雰囲気の参考画像 |
| 4 | なし（口頭で説明） | ※この場合、一緒にラフを作ります |

どれがありますか？（番号で選択、複数可）
```

**「4. なし」の場合:**
- ユーザーとの対話でレイアウト構造を決める
- ASCII やテキストベースのワイヤーフレームを提示して合意を取る
- 合意なしにコード実装に入らない

### 2. デザインシステムの定義

プロジェクトに既存のデザインシステムがあるか確認する:
- `tailwind.config.*`, `theme.*`, CSS 変数定義などを探す
- 既存があればそれに従う
- なければユーザーと一緒に定義する

**新規定義の場合、以下を順に決める:**

```
## デザインシステム定義

### カラーパレット
（ムードボードや参考画像から抽出、またはユーザーと相談）

| 役割 | カラー | 用途 |
|------|--------|------|
| Primary | #xxx | ボタン、リンク、主要アクション |
| Secondary | #xxx | 補助的な要素 |
| Accent | #xxx | 強調、バッジ |
| Background | #xxx | ページ背景 |
| Surface | #xxx | カード、モーダル背景 |
| Text | #xxx | 本文テキスト |
| Text Muted | #xxx | 補助テキスト |
| Border | #xxx | 境界線 |
| Error | #xxx | エラー状態 |
| Success | #xxx | 成功状態 |

### タイポグラフィ
- フォント: （Google Fonts から選定。Inter 以外を優先検討）
- 見出し: h1=2rem, h2=1.5rem, h3=1.25rem
- 本文: 1rem / 行間 1.6
- キャプション: 0.875rem

### スペーシング・角丸・シャドウ
- 基本単位: 4px
- 角丸: sm=4px, md=8px, lg=16px
- シャドウ: sm / md / lg の3段階

### アイコン
- ライブラリ: （Phosphor, Heroicons, Tabler Icons 等から選定）
```

ユーザーの承認を得てから次に進む。

### 3. コンポーネント構造の設計

ビジュアル参照をもとに、実装するコンポーネントの構造を提示する:

```
## コンポーネント構造

### ページ構成
- Header（ナビゲーション）
- Hero セクション
- ...

### 共通コンポーネント
- Button（primary / secondary / ghost）
- Card
- ...
```

構造の合意を得てから実装に入る。

### 4. 実装

以下のルールに従って実装する:

- デザインシステムの値をハードコードせず、変数・テーマ設定を使う
- 1コンポーネントずつ実装し、デザインシステムとの一貫性を確認する
- レスポンシブ対応を忘れない（モバイルファースト）
- 実装後、参考画像との差分をユーザーに確認してもらう

### 5. レビュー

実装完了後、以下のチェックリストを提示する:

```
## デザインレビューチェックリスト

- [ ] カラーパレットがデザインシステムに準拠している
- [ ] フォント・サイズが統一されている
- [ ] スペーシングが基本単位の倍数になっている
- [ ] アイコンが統一ライブラリから使用されている
- [ ] レスポンシブ表示を確認した（モバイル / タブレット / デスクトップ）
- [ ] 参考画像との差分を確認した
```

## 注意事項

- このワークフローは UI が関わる作業すべてに適用する（新規ページ、コンポーネント追加、デザインリニューアル等）
- 小さな修正（色の微調整、テキスト変更等）にはこのフローは不要
- デザインシステムが既に定義済みのプロジェクトではステップ2をスキップしてよい
