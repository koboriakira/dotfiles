# Overnight プロンプトテンプレート

生成するプロンプトは以下の構造に従う。

## テンプレート

```markdown
# 自律作業指示書

このプロンプトは非対話モードで実行される。ユーザーへの質問は一切できない。
判断に迷った場合は安全側に倒し、不明点はコミットメッセージやPR本文に記録すること。

## 共通ルール

- **ブランチ戦略**: 各イシューごとに `feat/<issue-number>-<short-description>` ブランチを作成する
- **コミット規約**: Conventional Commits に従う（feat:, fix:, refactor:, test:, docs:, chore:）
- **テスト必須**: 実装後に必ずテストを実行し、全テストが通ることを確認する
- **リンター必須**: プロジェクトにリンター設定があれば実行し、エラーを解消する
- **禁止事項**:
  - main/master への直接コミット・プッシュ
  - force push
  - 既存テストの削除・無効化
  - 機密ファイル（.env, credentials 等）の変更
  - package.json / Cargo.toml 等の依存関係の大幅な変更（パッチバージョン以外）
- **行き詰まった場合**: 無理に解決しようとせず、その時点までの成果をコミットし、PR本文に状況を記載する

## プロジェクト情報

- リポジトリ: {{owner}}/{{repo}}
- デフォルトブランチ: {{default_branch}}
- 言語/フレームワーク: {{language}}
- テストコマンド: {{test_command}}
- リンターコマンド: {{lint_command}}
- プロジェクト構造:
{{project_structure}}

## タスク一覧

### タスク 1: {{issue_title}} (#{{issue_number}})

#### 背景
{{issue_body}}

#### 関連コード
{{relevant_files}}

#### 受け入れ基準
{{acceptance_criteria}}

#### 作業手順
1. `feat/{{issue_number}}-{{short_desc}}` ブランチを作成
2. {{step_by_step_instructions}}
3. テストを追加・実行
4. リンターを実行
5. 変更をコミット
6. リモートにプッシュ
7. PR を作成（タイトル: "{{pr_title}}", 本文に作業内容と確認事項を記載）

#### 注意点
{{caveats}}

---

（タスク2以降、同じ構造で繰り返す）

## 完了報告

全タスク完了後、以下のサマリーを標準出力に出力すること:

### 作業結果サマリー
- タスク1: [完了/部分完了/未着手] — PR URL または状況
- タスク2: [完了/部分完了/未着手] — PR URL または状況
- ...

### 残課題
- （自律作業中に発見した問題や、手動対応が必要な事項）
```

## テンプレート利用時の注意

- `{{...}}` プレースホルダーは全て具体的な値で置換する
- イシュー本文が長い場合は要約する（ただし受け入れ基準は省略しない）
- 関連コードはファイルパスだけでなく、重要な型定義やインターフェースも含める
- 作業手順は具体的に書く（「適切に実装する」のような曖昧な表現は避ける）
- 1つのプロンプトに含めるタスクは3つ以下を推奨（多すぎるとコンテキスト不足になる）
