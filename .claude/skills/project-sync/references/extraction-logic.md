# 切り出しロジック詳細

## 判断フロー

```
入力: 前回実行以降に追加/更新されたセッションログとタスクボードエントリ

Step 1. リポジトリ単位でセッションをグルーピング

Step 2. 各リポジトリについて:

  Q1. 該当リポジトリにNotionのInProgressプロジェクトが既に存在するか？
    → Yes → 新規作成しない
    → No  → Q2へ

  Q2. ルーティン作業のみか？
    → Yes → プロジェクト化しない
    → No  → Q3へ

  Q3. テーマ的にまとまった作業群として言語化できるか？
    → No  → スキップ（次回の実行で再評価）
    → Yes → プロジェクトを作成する

Step 3. ステータス自動判定:
  - 最新セッションが2日以上前 かつ タスクボードの関連エントリがすべて完了済み → Done
  - それ以外 → InProgress
```

## ルーティン作業の判定基準

以下のパターンに合致するセッションはルーティンとして除外する:

### ファイル名パターン
- 「日次」「月次」「週次」を含む

### 繰り返しパターン
- 同一リポジトリで同一タイトルのセッションが複数日にわたって繰り返されている
- 例: `2026-03-01_日次調査.md`, `2026-03-02_日次調査.md`

### 既知のルーティン
| リポジトリ | パターン |
|-----------|---------|
| cloud-sourcing | 日次調査、日次スキャン |
| blog | 日次ブログ下書き生成 |
| create-service-import-minkan | 月次抽出 |
| screen-times | スクリーンOCRログ分析（定期実行分） |

## 重複防止ルール

1. 同一リポジトリ URL でステータスが InProgress のプロジェクトが存在する場合 → 新規作成しない
2. 同一リポジトリ URL でステータスが Done のプロジェクトと、名前が酷似するマイルストーン → 作成しない

### Notion での既存プロジェクト検索方法

`notion-search` でリポジトリ名をクエリとして検索し、結果の中からリポジトリ URL が一致するものをフィルタする。

## プロジェクト名の自動生成ルール

- 形式: `P:{リポジトリの文脈を踏まえたマイルストーン名}`
- セッションログの「概要」セクションからテーマを抽出して命名
- 複数セッションにまたがる場合は、共通テーマを抽象化

### 命名例

| セッション群 | プロジェクト名 |
|-------------|--------------|
| sandpiper 2/23-24 MCP関連 | P:sandpiper MCPサーバー構築＆Claude連携 |
| chat 2/27-28 Slack Bot関連 | P:Slack Bot MVP構築（Socket Mode + Claude Code CLI連携） |
| screen-times OCR基盤構築 | P:スクリーンOCR行動分析基盤構築 |
