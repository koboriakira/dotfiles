# Claude Code 使用量の最適化レビュー

現在の設定・セッション状態を診断し、トークン使用量の削減と効率化の具体的な提案を行う。

## 診断対象

以下を順に確認し、改善点があれば提案する。

### 1. CLAUDE.md の肥大化チェック
- `~/.claude/CLAUDE.md` のサイズ（目安: 5,000トークン以内が推奨）
- プロジェクト別の CLAUDE.md があれば同様にチェック
- `~/.claude/rules/` の各ファイルのサイズも確認
- 不要な重複・冗長な記述がないか

### 2. MCP サーバーとプラグインの棚卸し
- `~/.claude/settings.json` の `enabledPlugins` を確認
- 現在のプロジェクトで実際に使う言語/ツールに対して不要なプラグインがないか
- 有効な MCP サーバーの一覧を確認し、使っていないものを特定

### 3. モデル選択の確認
- 現在使用中のモデルを確認
- タスクの複雑さに対して適切なモデルか（簡単なタスクに Opus を使っていないか等）

### 4. 不要ディレクトリの除外設定
- `~/.claude/settings.json` と `.claude/settings.json` の `permissions.deny` を確認
- コンテキスト節約のために以下のような不要パスの `Read` が制限されているか
  - `node_modules/**`, `dist/**`, `build/**`, `.next/**` 等のビルド成果物
  - `*.lock` 等の巨大な自動生成ファイル
  - プロジェクト固有の大容量ディレクトリ
- 注: `.claudeignore` は未実装のため、`permissions.deny` が現時点の唯一の手段

### 5. コンテキスト管理の状況
- 会話の長さ・コンテキスト使用量が膨らんでいないか
- `/compact` や `/clear` の実行を推奨すべきタイミングか

## 出力フォーマット

診断結果を以下の形式で報告する:

```
## 診断結果

### 現状スコア: [A/B/C] （A=最適化済み、B=改善余地あり、C=要改善）

### 発見事項
- （具体的な問題点と数値）

### 推奨アクション
1. （優先度順に具体的な改善手順）
```
